Step 3: Download Dependencies.
Building and running ideal is easy on a Unix-like system (tested on Mac OS X and Linux.) The dependencies, besides the JDK, are JSR 305 annotations and JavaCUP. AppEngine SDK and Google Gson are needed if you want to build and run the Coach app.
Before running ideal, you need to create and populate the thirdparty directory. To do that and fetch all the dependencies, run bin/fetch-thirdparty.sh from the top-level directory. In addition, you'll need to create a symlink from thirdparty/jdk to the JDK home directory (such as /Library/Java/JavaVirtualMachines/jdk1.8.0_171.jdk/Contents/Home on Mac OS X.) It should be possible to invoke the Java compiler via thirdparty/jdk/bin/javac.
Step 4: Watch It Run!
Make is currently used to build and run ideal. Default target builds the create tool, bootstraps ideal.library.elements, analyzes it and prints it out. Targets hello and circle execute respective code from showcase directory. 12 executes tests/12.i. The targets generate_library and generate_runtime generate Java source code from library and runtime, respectively. The targets test_library and test_runtime test that the code compiles. The targets bootstrap_library and bootstrap_runtime bootstrap the code by overwriting the files in bootstrapped directory.
